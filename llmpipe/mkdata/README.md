# mkdata — Lexical data builder for nlpsolver

This directory builds synonym, antonym, and mutual-exclusion data files
consumed by the `llmpipe` reasoning pipeline. It is self-contained and
does not depend on any other `llmpipe` module.

The pipeline has two phases:

- **Cluster generation** (March 2026): fastText + WordNet clustering produces
  `syn_*_10.txt` and `ant_*.txt`.
- **Tier A/B pipeline** (April 2026): synonym harvesting, Policy C canonical
  selection, exclusion groups, and `build_solver_data.py` to generate the
  runtime dict files in `solver/`.

---

## (a) Main output files

These are the files meant to be consumed upstream (by the solver
integration, when wired up).

### Adjective synonyms — new two-tier pipeline

| File | Purpose |
|---|---|
| `syn_a_10.txt` | Adjective synonym cluster database (587 clusters as of this writing). Post-harvest, post-DROP state. Format A (`CID,member,score,...`). **Authoritative input**. |
| `syn_a_rewrite.txt` | Tier A hard-rewrite lookup: `member,canonical`. Generated by `pick_canonicals_a.py --emit`. Loaded by `logconvert.py` before clausification (not yet wired). |
| `syn_a_soft_axioms.txt` | Tier B soft biconditional axiom pair list: `word_a,word_b,score`. Generated by `pick_canonicals_a.py --emit`. Loaded by `lc_postprocess.py` at the end of `rawlogic_convert` (not yet wired). Merges UNSAFE-cluster canonical↔member pairs and the harvested sub-canonical stash. |
| `syn_a_canonicals_review.txt` | Human-readable Policy C classification report (SAFE / UNSAFE_WEIRD / UNSAFE_LOW / UNSAFE_NODROP with per-member DROPs). Review artifact, not consumed by the solver. |
| `syn_a_soft_pairs.txt` | Harvested sub-canonical pair stash (score 0.70–0.828, from `harvest_syn_a.py`). Merged into `syn_a_soft_axioms.txt` on emit. |

### Noun synonyms — Tier A/B pipeline (April 2026)

| File | Purpose |
|---|---|
| `syn_n_10.txt` | Noun synonym cluster database (1968 clusters, post-harvest, post-drops). Format A. |
| `syn_n_rewrite.txt` | Tier A hard-rewrite lookup: `member,canonical` (218 entries). |
| `syn_n_soft_axioms.txt` | Tier B soft biconditional pair list (6218 pairs). |
| `syn_n_canonicals_review.txt` | Human-readable Policy C classification report. |
| `syn_n_soft_pairs.txt` | Harvested sub-canonical stash (from `harvest_syn_n.py`). |

Noun Tier A filters: `MAX_MEMBER_NOUN_SYNSETS=3`, `MIN_MEMBER_SIMILARITY=0.88`,
plus 18-entry `BLOCKED_PAIRS` (proper nouns, religious terms, semantic mismatches).

### Verb synonyms — Tier A/B pipeline (April 2026)

| File | Purpose |
|---|---|
| `syn_v_10.txt` | Verb synonym cluster database (1093 clusters, post-harvest, post-drops). Format A. |
| `syn_v_rewrite.txt` | Tier A hard-rewrite lookup: `member,canonical` (124 entries). |
| `syn_v_soft_axioms.txt` | Tier B soft biconditional pair list (4103 pairs). |
| `syn_v_canonicals_review.txt` | Human-readable Policy C classification report. |
| `syn_v_soft_pairs.txt` | Harvested sub-canonical stash (from `harvest_syn_v.py`). |

Verb Tier A filters: `MAX_MEMBER_VERB_SYNSETS=5` (higher than nouns -- verbs
are more polysemous in WordNet), `MIN_MEMBER_SIMILARITY=0.88`.

### Antonyms (all POS)

| File | Purpose |
|---|---|
| `ant_a.txt` | Adjective antonym pairs (~1071 total). Format A. Sources: WordNet + manual + external dataset merges (applied historically, merge scripts removed). |
| `ant_n.txt` | Noun antonym pairs. WordNet only. |
| `ant_v.txt` | Verb antonym pairs. WordNet only. |

`build_antonyms` in `build_solver_data.py` applies two symmetric guards
against chain-contamination with `CANONICALS`:
1. `word in CANONICALS` — skip (canonical pass would shadow the fold source).
2. `canonical in CANONICALS` — skip rewriting (canonical pass would
   chain-substitute the fold target to an unrelated sense). Rejected pairs
   are deferred to `build_exclusions` and emitted as defeasible adjective
   mutual-exclusion groups `ANT_<W1>_<W2>` instead. Typical cause: WordNet
   verb↔prep/adj homograph (e.g. `open ↔ close` verb-sense combined with
   `CANONICALS["close"] = "near"` adj-sense).

### Adjective exclusion groups

| File | Purpose |
|---|---|
| `excl_a.txt` | Mutual-exclusion groups for adjectives (~60 groups). Format: `GROUP_ID,source,score,needs_blocker,word1,...,wordN`. The `needs_blocker` flag is 0 or 1: 1 means the cluster needs a guard predicate (e.g. `multicolored` for COLOR); 0 means exclusivity is near-absolute (months, classes, directions). `build_solver_data.py` augments this with synthetic `MANUAL_ADJ_*` groups (from `MANUAL_ANTONYMS`) and `ANT_*` groups (from chain-rejected antonym pairs) — total ~123 groups in the runtime `data_exclusions.py`. |

### Solver runtime data (generated by `build_solver_data.py`)

| File | Purpose |
|---|---|
| `solver/data_canonicals.py` | ~752 Tier A `{variant: canonical}` entries (all POS merged). Used by `semnormalize.py`. |
| `solver/data_antonyms.py` | ~850 `{word: antonym}` directional pairs. Used by `semnormalize.py`. Pairs whose target is itself a CANONICALS key are dropped here and deferred to `data_exclusions.py` as synthetic `ANT_*` groups (prevents chain-substitution in semnormalize Pass 2 from producing nonsense atoms, e.g. `open→close→near`). |
| `solver/data_synonyms.py` | ~12K words, bidirectional `{word: [(other, score, pos), ...]}`. Used by `lc_postprocess.py`. |
| `solver/data_exclusions.py` | ~123 groups + inverted index (~441 indexed words). Used by `lc_postprocess.py`. Includes `excl_a.txt` groups plus synthetic `MANUAL_ADJ_*` (from `MANUAL_ANTONYMS`) and `ANT_*` (from chain-rejected antonym pairs). |

Regenerate after any source .txt changes: `cd mkdata && python3 build_solver_data.py`

## (b) Pipelines and software

Each pipeline below lists its script, inputs, and outputs.

### 1. Cluster build (original, March 2026)

```
cc.en.300.bin (fastText) + WordNet (NLTK) + wordfreq
             │
             ▼
    make_anto_synonyms.py
             │
             ▼
  syn_{a,n,v}_10.txt + ant_{a,n,v}.txt
```

fastText cosine similarity + WordNet Wu-Palmer blending, filtered by
`--min_zipf 3.5`. ~10–30 min per POS. Produces the baseline clusters.

### 2. Exclusion groups (April 2026)

```
WordNet synset.attributes() ────┐
                                │
chromatic_color.n.01 hyponyms ──┤
(currently unused — stub)       │
                                ├──► build_exclusion_data.py ──► excl_a.txt
manual categorical taxonomies ──┤                                ant_{a,n,v}.txt
(32 curated groups)             │
                                │
WN_ATTRIBUTE_KEEP allowlist ────┘
(5 curated synset names)
```

Run with `--only_excl` to rebuild just `excl_a.txt` without clobbering
`ant_*.txt` (which have been merged with external sources).

### 3. Synonym harvest (April 2026)

```
zaibacu en_thesaurus.jsonl (mkdata/roget.json)
            │
            ▼
  harvest_syn_a.py  ──►  syn_a_10.txt          (572 → 587 clusters, --apply)
            │            syn_a_soft_pairs.txt  (44 sub-canonical pairs)
            │            syn_a_additions_review.txt
            │
     uses:  cc.en.300.bin (fastText cosine scoring)
            WordNet (sense-alignment filter, same-synset / similar_tos)
            wordfreq (zipf filter, canonical pick)
            syn_a_10.txt (ADD target lookup, canonical_set protection)
```

Classifies every new candidate pair as ADD (one word in a cluster),
NEW (both absent), or MERGE (both in different clusters, skipped).
Applies a Policy-C-style primary-sense alignment filter, then fastText
scores. Thresholds: ADD members ≥ 0.70, NEW canonical-worthy ≥ 0.828,
NEW 0.70–0.828 → soft-pair stash.

### 4. Canonical selection — Policy C (April 2026)

```
syn_{a,n,v}_10.txt ──► pick_canonicals_{a,n,v}.py ──► syn_*_canonicals_review.txt
                              │                        syn_*_10.txt  (drops with --apply)
                              │                        syn_*_rewrite.txt    (with --emit)
                              │                        syn_*_soft_axioms.txt (with --emit)
                              │
                       uses:  WordNet, wordfreq, canonical_set, soft_pairs
```

Classifies each cluster as SAFE / UNSAFE_WEIRD / UNSAFE_LOW / UNSAFE_POS /
UNSAFE_NODROP. Winner is always the current CID canonical (never replaced).
For SAFE clusters, per-member drops are computed against primary-sense
alignment, with canonical-set protection (rule 5) and UNSAFE_NODROP fallback.

Noun and verb variants add polysemy and similarity gates beyond the base
adjective thresholds:
- Nouns: `MAX_MEMBER_NOUN_SYNSETS=3`, `MIN_MEMBER_SIMILARITY=0.88`, `BLOCKED_PAIRS` (18 entries)
- Verbs: `MAX_MEMBER_VERB_SYNSETS=5`, `MIN_MEMBER_SIMILARITY=0.88`

### 5. Solver data generation (April 2026)

```
syn_{a,n,v}_rewrite.txt ──┐
ant_{a,n,v}.txt ───────────┤
syn_{a,n,v}_soft_axioms.txt┤──► build_solver_data.py ──► solver/data_canonicals.py
excl_a.txt ────────────────┘                              solver/data_antonyms.py
                                                          solver/data_synonyms.py
                                                          solver/data_exclusions.py
```

Fast step (~1 sec). Reads all mkdata .txt sources, generates Python dict files
in `solver/`. Must be re-run after any source .txt changes:

```bash
cd mkdata && python3 build_solver_data.py
```

The generated files are loaded at import time by `semnormalize.py` and
`lc_postprocess.py`. See DOCUMENTATION.md section 9 for integration details.

### Candidate future sources (explored and rejected)

- **PPDB 2.0**: Too large and noisy for the current pipeline; not yet pursued.
- **Real Roget 1911 (Project Gutenberg etext)**: Not pursued; majority of its vocabulary fails `zipf ≥ 4.0` due to archaic terms.
- **WordNet hyponyms of `material.n.01`, `fabric.n.01`, `metal.n.01`, `wood.n.01`, `shape.n.02`**: Heavily contaminated by noun-homonym adjective matches. Abandoned.
- **`natural_language.n.01` hyponyms**: Missed major languages (English, French) buried deep in family branches. Replaced with a manual LANGUAGE list.

## (d) Methods and observations

Ordered by pipeline. "Worked" means the method produced usable output
with acceptable noise; "partially" means some manual review was needed;
"failed" means we abandoned it.

### Synonym cluster build (`make_anto_synonyms.py`)
- **fastText cosine + WordNet blending**: **worked**. Produced baseline `syn_*_10.txt` that the rest of the pipeline rests on. Key tuning: `--min_zipf 3.5`, `--require_min_syn 3`, relational-adjective filter enabled.

### Antonym KB enrichment (historical, merge scripts removed)
The current `ant_a.txt` includes ~437 extra pairs merged from three external
sources (public CSV, hand-curated lists, EACL 2017 benchmark) on top of the
634 WordNet-only pairs. The merge scripts have been removed; their results
are baked into `ant_a.txt`.

### Synonym harvest (`harvest_syn_{a,n,v}.py`)
- **Source labeled Roget 1911 was actually WordNet**: the zaibacu/thesaurus JSONL has `wordnet_id` fields on every entry — it is a repackaged WordNet, not Roget. The project memory initially misidentified it. **Partial signal** because `syn_a_10.txt` used fastText+threshold while zaibacu lists all WordNet synonym edges — the diff is exactly the pairs that failed the old fastText gate.
- **Sense-alignment + WordNet-strong filter + fastText cosine scoring**: **worked**. 19,558 raw novel pairs → 545 WordNet-strong at zipf ≥ 4.0 → 67 ADD + 119 NEW after primary-sense alignment → 27 applied as members + 15 new clusters + 44 soft pairs at the user's 0.70 / 0.828 thresholds.
- **Multi-CID duplication**: the same word sometimes lands in 3–4 sibling CIDs (satellite clusters of the same head synset). Added to all. Not deduped.

### Policy C canonical selection (`pick_canonicals_a.py`)
- **POS-weighted zipf for winner gate** (`adj_zipf = zipf * adj_synsets / total_synsets`): **partial**. Original `MIN_ADJ_ZIPF = 3.0` killed many legitimate core adjectives because of cross-POS polysemy (right/free/high/far). Proposed fix (raw `zipf ≥ 4.0` for winner) is still deferred.
- **Effective-primary sense group with lonely-head skip and satellite-group expansion**: **worked** after the satellite bug was fixed. Original expansion only followed `primary.similar_tos()`, missing sister satellites; now resolves through the head synset correctly.
- **Canonical-set protection (rule 5)**: **worked**. Members that are themselves canonicals of any cluster are never dropped and never emitted as rewrite sources.
- **UNSAFE_NODROP safety fallback**: **worked**. Three clusters (`ALIVE_A02`, `IGNORANT_A01`, `ALTERNATIVE_AH01`) would have been fully emptied under a strict reading — the fallback leaves them intact.
- **Fixed-point instability**: running Policy C twice (once before `--apply`, once after) produces slightly different DROP sets because removing a member shrinks the "other cluster words" anchor set for the remaining members. Mild (8 drops appeared on the second pass). Not automated to fixed point yet.

### Exclusion groups (`build_exclusion_data.py`)
- **`synset.attributes()` extraction**: **largely failed**. 118 of 173 output groups were pure 2-element antonym pairs; most 3+ groups were 2+2 antonym+synonym mixtures (CORRECTNESS: right/wrong/correct/incorrect) or synonym-only (CYCLICITY: cyclic/cyclical). Salvaged by switching to a strict `WN_ATTRIBUTE_KEEP` allowlist of 5 synset names.
- **WordNet noun-hyponym walks for exclusion**: **failed**. `material.n.01`, `fabric.n.01`, `metal.n.01`, `shape.n.02` all produced heavy noise from adjective homonyms of unrelated noun senses (`material.n.01` pulled in `color`, `orange`, `ruby`, `salt`). `natural_language.n.01` missed major languages and pulled in family/region names. The `NOUN_CATEGORY_ROOTS` extraction path is kept as a stub but currently has zero entries.
- **Manual categorical curation**: **worked**. 35 of the 40 final groups are manual. Division into `needs_blocker=0` (hard exclusive: months, days, classes, directions) and `needs_blocker=1` (composite states common: colors, materials, weather, tastes) made per-group.
- **Web research for new categories**: **worked**. 10 new groups added from Wikipedia list pages (weather, cooking, government, rock types, animal class, art movements, architectural styles, patterns, educational levels, musical tempos).
- **Kaikki Wiktextract `coordinate_terms` mining**: **partial**. 576 raw clusters with ≥3 coord_terms → 71 survived syn/ant filtering → only ~8 looked like genuine forced-choice taxonomies. Real-signal rate ~1.4% of raw candidates. Not worth automating, but mined manually for 3 additions (FREQUENCY, CHRISTIAN_DENOMINATION, GROWTH_RATE).

### Meta-observations
- WordNet's structural relations (`attributes`, `hyponyms`, `similar_tos`) are not well-aligned with "mutually exclusive category" as a concept. Antonyms, synonyms, and category-siblings all bleed into each other.
- Manual lists remain the highest-precision source for exclusion groups. Automated sources are best treated as candidate feeders for manual review.
- Fastrack pipeline idea that did NOT work: "use the count of WordNet antonym pairs inside a group to auto-detect 2+2 antonym structures". Some real scale-position groups like TEMPERATURE (hot/cold/warm/cool) are also 2+2 antonym structures and would be false-positively dropped.

## Cluster file format (Format A)

Unchanged from the original pipeline. The `syn_*_10.txt` and `ant_*.txt`
files use:

```
CANONICAL_ID,word1,score1,word2,score2,...
```

- `CANONICAL_ID` = `LEMMA_POS##` (e.g. `FAST_A01`, `CAR_N01`, `THINK_V01`)
- The canonical lemma is implicit in the ID and is **not** listed as a
  member — only the non-canonical synonyms appear.
- Polysemous words get multiple clusters (`WAY_N01`, `WAY_N02`, ...).
- Scores are in [0, 1], sorted descending.
- Provenance suffixes on the CID digits distinguish sources in merged
  files: `_A01` (original WordNet), `_AM01` (manual), `_AC01` (CSV),
  `_AV01` (EACL val), `_AH01` (harvested from zaibacu).

## Environment setup

All scripts run inside `mkdata/venv/`. The venv is not committed.

```bash
cd mkdata/
python3 -m venv venv
venv/bin/pip install wordfreq fasttext nltk numpy
venv/bin/python -c "
import nltk
nltk.download('wordnet')
nltk.download('omw-1.4')
"
```

For `make_gradables.py`:
```bash
venv/bin/pip install spacy
venv/bin/python -m spacy download en_core_web_sm
```

Always prefix commands with `venv/bin/python`.
