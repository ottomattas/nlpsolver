CRITICAL RULES — VERIFY BEFORE FINALIZING OUTPUT:

1. WH-QUESTIONS USE "ask" (§7.1): If any entity has wh_placeholder: true,
   the ASU MUST use ["ask", VAR, BODY], never ["question", ...].

2. GENERIC ENTITIES → VARIABLES (§4.2): Every generic entity (except scope
   "kind") must introduce a fresh variable. Never use a generic ID as a
   ground constant in the logic.

3. STATIVE VERBS ARE NOT EVENTS (§6): have, own, like, love, know, believe,
   contain — encode as direct predicates, never as event reification.

4. has_degree_property IFF IN ADJECTIVES (§2.5): Use has_degree_property
   only when the word appears in ASU.adjectives. Use has_property otherwise.
   Never emit both for the same word.

5. NO "is" AS RELATION (§5.7): Never use ["is rel2", "is", ...]. Use
   ["isa", TYPE, ENTITY] for category, ["=", A, B] for identity.

6. FLAT ANTECEDENT (§5.8): Never nest ["implies", A, ["implies", B, C]].
   Flatten to ["implies", ["and", A, B], C].

7. POSITIVE FORMULA FOR LOW PROBABILITY (§5.9): "rarely" / "hardly" →
   positive formula + ["@p", "Sx", low_value]. Never use ["not",...] for
   frequency adverbs.

8. NO TENSE IN has_time (§8.1): has_time takes explicit time values only
   ("1800", "Monday"). Never put "past"/"present"/"future" in has_time.

9. ISA FOR CONCRETE OBJECTS (§4.4): Every concrete entity in a situation
   must have an ["isa", TYPE, ID] predicate in the same ASU.

10. ENTITY IDS FROM STAGE 1 (§4.1): Use entity IDs exactly as given by
    Stage 1. Do not rename, renumber, or drop qualifiers from IDs.

11. ["typical","E"] LAST (§6.3): In Track 2 event blocks, ["typical","E"]
    must always be the last conjunct inside the "and".

12. ONE @id PER ASU (§0.3): Each ASU gets exactly one ["@id","Sx",...]
    wrapper. No nesting, no duplication.
