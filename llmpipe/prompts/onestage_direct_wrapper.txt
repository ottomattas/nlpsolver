== ONE-STAGE DIRECT MODE ==

You convert English text DIRECTLY into first-order-logic JSON in a SINGLE step.

Normally this conversion runs in two stages: (1) English is annotated into
Atomic Semantic Units (ASUs), then (2) the ASUs are converted to logic JSON.
Here you do BOTH at once. The two full specifications are given below AS
REFERENCE KNOWLEDGE so you know the entity-id, segmentation, and logic-encoding
conventions — but you apply them in one step.

YOUR INPUT: raw English text (statements and possibly a question).
YOUR OUTPUT: ONLY the final logic JSON list — the exact format produced at the
END of stage 2. Do NOT output ASUs, reasoning, explanations, or any other text.

You must still apply every Stage-1 convention internally (segment the text into
units S1, S2, ...; assign entity ids like "John 1"; mark questions) and every
Stage-2 logic rule — but emit ONLY the final logic list.

OUTPUT CONTRACT (identical to Stage-2):

  ["and", ["@id","S1", PACKAGE], ["@id","S2", PACKAGE], ...]

  - Output exactly ONE JSON list. No JSON objects. No commentary.
  - One ["@id","Sx", PACKAGE] per sentence/unit, numbered S1, S2, ...
  - Non-question package: ["holds", W, F]   Question: ["question",...] or ["ask","X",...]
  - No invented predicates. All variables bound. No free variables.

EXAMPLE:

  INPUT:  John is a person.
  OUTPUT: ["and",["@id","S1",["holds","W0",["isa","person","John 1"]]]]

The two reference specifications follow.
