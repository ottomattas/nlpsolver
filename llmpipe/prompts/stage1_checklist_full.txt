CRITICAL RULES — VERIFY BEFORE FINALIZING OUTPUT:

1. ONE QUERY PER QUESTION (§8.1): A sentence ending with "?" produces EXACTLY ONE ASU
   with "type": "query". Never split a question into multiple query ASUs.

2. SAME NAME = SAME ID (§4.4): Reuse the same entity ID for the same name across all
   sentences. "Mary" in sentence 1 and "Mary" in sentence 3 are both "Mary 1".

3. DIFFERENT INDEFINITES = DIFFERENT ENTITIES (§4.1): "A bear was strong. A bear was
   nice." — these are two different bears (bear 1 and bear 2), not one.

4. INDEFINITES IN QUERIES ARE GENERIC (§4.2): Indefinite nouns in questions ("a car",
   "a trunk") must be type: "generic", never concrete with numbered IDs.

5. BARE PLURAL QUESTIONS → "ALL" (§8.3): "Elephants are animals?" must be normalized
   to "Are all elephants animals?" — not treated as existential.

6. PRESERVE NEGATION (§8.4): Never drop "no", "not", "none", "nothing", "never" when
   reformulating questions or statements.

7. DO NOT SPLIT OR / IF-THEN (§2.1): Sentences with "or", "either...or", "if...then",
   "unless" stay as one ASU.

8. CATEGORY NOUNS ARE NOT ADJECTIVES (§7.1): Words like "animal", "bird", "person",
   "toy" go in entities, never in the adjectives list.

9. KEEP QUESTION TENSE (§8.6): If the question uses past tense ("was", "did", "had"),
   the query ASU must have "time": "past".

10. UNIQUE SEQUENTIAL UNIT IDS (§1.1): S1, S2, S3... — never reuse a unit_id.

11. STATIVE VERBS STAY IN CURRENT WORLD (§9): "is", "has", "likes", "knows" do NOT
    create a new world state. Only movement/transfer/change-of-state events do.

12. NO SETS FOR ORDINARY PLURALS (§4.3): "the elephants", "they", "the cars" are
    simple entities, not set entities — unless the text explicitly counts or
    enumerates a group.
